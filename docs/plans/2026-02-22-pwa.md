# PWA Support Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add PWA support to ServerMonitor so users can install it to their home screen/desktop and have the app shell load from cache when offline.

**Architecture:** Use `vite-plugin-pwa` which generates a Workbox-powered service worker at build time and auto-injects the web app manifest. Static assets use cache-first; API routes use network-first. No manual SW file. All config in `vite.config.ts`.

**Tech Stack:** Vite 5, Vue 3, vite-plugin-pwa, Workbox (bundled with plugin)

### Task 2: Generate PWA icons from favicon.svg

**Files:**
- Create: `frontend/public/icons/pwa-192x192.png`
- Create: `frontend/public/icons/pwa-512x512.png`

The existing favicon is `frontend/public/favicon.svg` — a monitor icon with a blue stroke on transparent background.

**Step 1: Check if sharp or Inkscape is available**

```bash
which inkscape || which rsvg-convert || which convert
```

**Step 2a: If ImageMagick (`convert`) is available**

```bash
mkdir -p frontend/public/icons

# 192x192 on dark background
convert -background "#0f172a" -density 192 frontend/public/favicon.svg \
  -resize 192x192 frontend/public/icons/pwa-192x192.png

# 512x512 on dark background
convert -background "#0f172a" -density 512 frontend/public/favicon.svg \
  -resize 512x512 frontend/public/icons/pwa-512x512.png
```

**Step 2b: If Inkscape is available**

```bash
mkdir -p frontend/public/icons
inkscape frontend/public/favicon.svg -w 192 -h 192 -o frontend/public/icons/pwa-192x192.png
inkscape frontend/public/favicon.svg -w 512 -h 512 -o frontend/public/icons/pwa-512x512.png
```

**Step 2c: If neither is available — use Node script**

Create a temporary file `frontend/gen-icons.mjs`:
```js
import { execSync } from 'child_process'
import { mkdirSync } from 'fs'

mkdirSync('public/icons', { recursive: true })
// Uses npx sharp-cli if available
execSync('npx sharp-cli --input public/favicon.svg --output public/icons/pwa-192x192.png resize 192 192')
execSync('npx sharp-cli --input public/favicon.svg --output public/icons/pwa-512x512.png resize 512 512')
```

Then run: `cd frontend && node gen-icons.mjs` and delete the script afterwards.

**Step 3: Verify icons exist and are non-zero**

```bash
ls -lh frontend/public/icons/
```

Expected: two PNG files, each > 1KB.

**Step 4: Commit**

```bash
git add frontend/public/icons/
git commit -m "feat(pwa): add PWA icon assets (192x192, 512x512)"
```

**Step 1: Replace the file with PWA config added**

```ts
import { fileURLToPath, URL } from 'node:url'
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import { VitePWA } from 'vite-plugin-pwa'

export default defineConfig({
  plugins: [
    vue(),
    VitePWA({
      registerType: 'autoUpdate',
      manifest: {
        name: 'ServerMonitor',
        short_name: 'ServerMonitor',
        description: 'Homelab server monitoring dashboard',
        theme_color: '#0f172a',
        background_color: '#0f172a',
        display: 'standalone',
        scope: '/',
        start_url: '/',
        icons: [
          {
            src: '/icons/pwa-192x192.png',
            sizes: '192x192',
            type: 'image/png'
          },
          {
            src: '/icons/pwa-512x512.png',
            sizes: '512x512',
            type: 'image/png',
            purpose: 'any maskable'
          }
        ]
      },
      workbox: {
        // Cache-first for app shell assets
        globPatterns: ['**/*.{js,css,html,ico,svg,png,woff2}'],
        // Network-first for API routes with cache fallback
        runtimeCaching: [
          {
            urlPattern: /^https?:\/\/.*\/api\/.*/i,
            handler: 'NetworkFirst',
            options: {
              cacheName: 'api-cache',
              networkTimeoutSeconds: 10,
              expiration: {
                maxEntries: 50,
                maxAgeSeconds: 60 * 5  // 5 minutes
              },
              cacheableResponse: {
                statuses: [0, 200]
              }
            }
          }
        ]
      }
    })
  ],
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url))
    }
  },
  server: {
    port: 4829,
    proxy: {
      '/api': { target: 'http://localhost:8742', changeOrigin: true },
      '/ws': { target: 'ws://localhost:8742', ws: true }
    }
  }
})
```

### Task 6: Add offline indicator to the UI (optional but recommended)

**Files:**
- Modify: `frontend/src/App.vue`

When the user is offline, show a subtle banner so they know the data is stale.

**Step 1: Add the offline banner to App.vue**

Read the current `App.vue` first, then add this inside the template (just after the opening `<div>` or before the `<RouterView>`):

```vue
<div
  v-if="!isOnline"
  class="fixed top-0 inset-x-0 z-50 bg-amber-500 text-black text-center text-sm py-1 font-medium"
>
  Offline — showing cached app shell
</div>
```

And in `<script setup>`:

```ts
import { useOnline } from '@vueuse/core'

const isOnline = useOnline()
```

Wait — `@vueuse/core` may not be installed. Check first:

```bash
grep vueuse frontend/package.json
```

If not installed, implement without the library:

```ts
import { ref, onMounted, onUnmounted } from 'vue'

const isOnline = ref(navigator.onLine)
const handleOnline = () => { isOnline.value = true }
const handleOffline = () => { isOnline.value = false }

onMounted(() => {
  window.addEventListener('online', handleOnline)
  window.addEventListener('offline', handleOffline)
})
onUnmounted(() => {
  window.removeEventListener('online', handleOnline)
  window.removeEventListener('offline', handleOffline)
})
```

**Step 2: Verify the banner appears when offline**

In DevTools → Network → check Offline → the amber banner should appear at the top of the page.

**Step 3: Commit**

```bash
git add frontend/src/App.vue
git commit -m "feat(pwa): add offline status banner"
```
